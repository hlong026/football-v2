# 二次清洗与送模链路需求说明

## 1. 背景

当前系统已经具备以下能力：

- 前端接收用户输入的比赛链接、时间范围、欧赔机构、亚盘机构
- 后端完成页面采集与一次结构化解析
- 结构化结果被直接拼装后传给 AI 模型分析

本次改造目标是在“一次结构化解析”和“AI 模型分析”之间新增“二次清洗层”，并将原始结构化数据与清洗后数据都以分层文件方式落盘。

## 2. 总体目标

系统在完成欧赔与亚盘数据采集后，需要结合用户前端输入的时间范围与机构范围，对采集结果进行同步二次清洗。

改造后的链路如下：

`前端输入 -> 页面采集 -> 一次结构化 -> 二次清洗 -> 清洗结果输出 -> AI 分析`

其中：

- `一次结构化` 的输出属于清洗前数据
- `二次清洗` 的输出属于清洗后数据
- `AI 分析` 阶段只允许使用清洗后的分层数据

## 3. 用户输入条件

二次清洗必须同步结合以下用户输入条件：

- 指定时间范围
  - 起始时间 `anchor_start_time`
  - 结束时间 `anchor_end_time`
- 指定机构范围
  - 欧赔机构列表
  - 亚盘机构列表

## 4. 时间处理规则

时间处理采用统一标准：

- 前端输入格式：`datetime-local`
- 前端传输格式：ISO 8601
- 后端比较与存储：统一为 UTC
- 后端展示与输出：统一提供北京时间 `Asia/Shanghai`

过滤规则：

- 仅保留命中时间窗口的数据
- 过滤条件为 `start <= change_time <= end`
- 当结束时间为空时，表示从开始时间到当前采集范围末尾
- 当结束时间早于开始时间时，视为非法请求

## 5. 机构处理规则

机构处理采用“用户指定机构 + 机构名称归一化 + 命中筛选”的方式。

规则包括：

- 对用户输入机构名称进行拆分、去重与标准化
- 结合系统内置别名规则完成机构名称归一化
- 仅保留命中的欧赔机构与亚盘机构
- 输出机构映射信息，至少包含：
  - 用户输入机构名
  - 归一化机构名
  - 实际命中机构名
  - 是否命中

## 6. 欧赔清洗规则

欧赔数据清洗必须结合“时间范围”和“机构范围”同步执行。

清洗范围：

- 仅保留用户指定欧赔机构
- 仅保留用户指定时间范围内的欧赔变化记录

字段规则：

- 删除首行数据记录
- 删除 `更新` 列
- 删除 `指数` 列整组内容，即：
  - 主
  - 平
  - 客
- 不保留页面头部、导航、按钮等无关内容

欧赔最终保留字段：

- `时间`
- `概率(主/平/客)`
- `凯利指数(主/平/客)`
- `赔付率`

## 7. 亚盘清洗规则

亚盘数据清洗必须结合“时间范围”和“机构范围”同步执行。

清洗范围：

- 仅保留用户指定亚盘机构
- 仅保留用户指定时间范围内的亚盘变化记录

字段规则：

- 删除 `更新` 列
- 不采集水位走势图
- 不采集盘口走势图
- 不保留页面头部、导航、按钮等无关内容

亚盘最终保留字段：

- `时间`
- `主队水位`
- `盘口`
- `客队水位`

## 8. 数据落盘要求

系统必须在 `backend/data` 目录下，以分层结构同时保存清洗前和清洗后数据。

推荐目录结构如下：

```text
backend/data/pipeline/
  YYYY-MM-DD/
    site/
      match_key/
        run_id/
          raw-structured.json
          cleaned-structured.json
          analysis-input.json
          analysis-output.json
          meta.json
```

文件含义：

- `raw-structured.json`
  - 一次结构化后的原始数据
  - 属于清洗前数据

- `cleaned-structured.json`
  - 结合时间与机构规则清洗后的分层数据
  - 属于后续业务主数据

- `analysis-input.json`
  - 实际传给 AI 模型的最终数据
  - 必须使用清洗后数据

- `analysis-output.json`
  - AI 模型分析结果

- `meta.json`
  - 记录任务元信息
  - 至少应包含站点、比赛 ID、源链接、时间范围、机构范围、创建时间

## 9. 送模规则

最终传给模型的数据必须满足以下规则：

- 只允许使用清洗后的分层数据
- 不允许直接使用清洗前的原始结构化数据
- 模型输入预览展示的内容，也必须是最终送模数据

## 10. 接口与类型改造要求

建议新增或改造以下能力：

- 新增二次清洗服务
  - 输入：一次结构化结果 + 用户输入时间 + 用户输入机构
  - 输出：清洗后的分层数据

- 新增清洗结果输出接口
  - 用于单独查看二次清洗后的结果

- 改造模型输入预览接口
  - 预览内容切换为最终 cleaned payload

- 改造正式分析接口
  - 模型调用时只传 cleaned payload

## 11. 验收标准

以下条件全部满足，视为本次需求完成：

- 用户指定时间范围后，仅输出对应时间窗口内的数据
- 用户指定机构范围后，仅输出对应机构的数据
- 欧赔数据已删除首行、更新列、指数列，仅保留时间、概率、凯利、赔付率
- 亚盘数据仅保留时间与主盘数据，不包含更新列与走势图内容
- `data` 目录下同时存在清洗前和清洗后分层数据
- `analysis-input.json` 与模型预览内容一致
- 正式模型分析仅使用清洗后数据
